# MAYO-2 Oracle Fixtures

These vectors are generated by `tools/mayo2-oracle`.

```sh
cargo run --manifest-path tools/mayo2-oracle/Cargo.toml -- check-shake-fixture
cargo run --manifest-path tools/mayo2-oracle/Cargo.toml -- check-pq-synthetic-fixture
cargo run --manifest-path tools/mayo2-oracle/Cargo.toml -- check-pq-mayo-fixture
```

- `shake-synthetic-oracle.json` is the scalar SHAKE verify-map vector currently
  mirrored by `motoko/test/Mayo2ShakeVerifier.test.mo`.
- `pq-synthetic-oracle.json` is the scalar production `pq-mayo` verify-map
  vector mirrored by `motoko/test/Mayo2PqVerifier.test.mo`.
- `pq-mayo-consensus-sample.json` is the production-style `pq-mayo` sample:
  `186`-byte signature, `4,912`-byte compact public key, and
  `mayo-consensus` domain separation.

The SHAKE fixture proves the Motoko port's field/hash/verify-map mechanics. The
synthetic `pq-mayo` fixture proves the production target hash, entry-major
matrix layout, and `compute_rhs` pair order. The full compact `pq-mayo` fixture
is pinned with bounded and long Motoko gates:

```sh
mops test Mayo2PqRealVector
mops test Mayo2PqFullVerify
```

`Mayo2PqRealVector` verifies, for all three real vectors:

- compact public-key and signature sizes,
- `pq-mayo` target nibbles,
- AES-128-CTR P1/P2 expansion head and tail windows.

`Mayo2PqFullVerify` verifies all three real compact keys/signatures end to end
through `verifyCompactBytes`. On 2026-06-24 it passed under the Motoko
interpreter in 76.69 seconds after AES-CTR and packed GF(16) hot-loop
optimization. The remaining production gate is a compiled canister/cycle
benchmark for one `verifyCompactBytes` call, plus real-vector mutation/negative
coverage.
